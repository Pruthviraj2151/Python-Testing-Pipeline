## Architecture

```
Developer → Git Push → GitHub → Webhook → Jenkins
                                              │
              ┌───────────────────────────────┤
              │                               │
         Code Quality                  Deployment
              │                               │
     Lint → Test → Coverage           Docker Build
              │       │                      │
          JUnit XML  HTML Report       Container Run
              │                               │
         SonarQube Analysis            Health Check
              │                               │
         Quality Gate ──────── Pass ──── Dashboard
                     └── Fail → Email Alert
```

## Quick Start

### Prerequisites

- Python 3.12+
- Docker & Docker Compose
- Jenkins (with Blue Ocean, HTML Publisher, Email Extension plugins)
- SonarQube (optional — pipeline skips gracefully if absent)

### 1. Clone the repository

```bash
git clone https://github.com/yourusername/Python-Testing-Pipeline.git
cd Python-Testing-Pipeline
```

### 2. Run tests locally

```bash
python -m venv .venv
source .venv/bin/activate          # macOS/Linux
# .venv\Scripts\activate           # Windows

pip install -r requirements.txt
pytest
```

### 3. Run with Docker

```bash
docker-compose up --build
```

App available at `http://localhost:5000`

### 4. View Dashboard

Open `dashboard/index.html` in your browser (or serve via any static file server).

## Pipeline Stages

| # | Stage | Description |
|---|-------|-------------|
| 1 | Checkout | Pulls latest source from GitHub |
| 2 | Setup Env | Creates Python virtual environment |
| 3 | Install | Installs all pip dependencies |
| 4 | Lint | Flake8 PEP-8 compliance check |
| 5 | Test | Pytest with 180+ tests |
| 6 | Coverage Gate | Fails pipeline if coverage < 80% |
| 7 | SonarQube | Static analysis scan |
| 8 | Quality Gate | Blocks on SonarQube quality gate |
| 9 | Docker Build | Builds tagged Docker image |
| 10 | Deploy | Replaces running container |
| 11 | Health Check | Polls `/health` until app is ready |
| 12 | Archive | Stores all reports as Jenkins artifacts |
| 13 | Dashboard | Updates `build-data.json` for live dashboard |

## Jenkins Setup

1. Install Jenkins plugins: Blue Ocean, HTML Publisher, Email Extension, Git, Docker Pipeline, SonarQube Scanner
2. Add credentials: `NOTIFICATION_EMAIL`, `SONAR_TOKEN`
3. Configure SonarQube server in Jenkins → Manage Jenkins → Configure System
4. Create a new Pipeline job, point to this repo, and set `Jenkinsfile` as the pipeline script
5. Add GitHub webhook: `http://your-jenkins:8080/github-webhook/`

## Live Demo Sequence

**Scenario 1 — Successful Build**
```bash
git add . && git commit -m "feat: add statistics functions" && git push
# Watch: Pipeline runs green, Docker deploys, dashboard shows SUCCESS
```

**Scenario 2 — Failed Build**
```python
# Break calculator.py: change `a + b` to `a - b` in add()
git add . && git commit -m "bug: broken add" && git push
# Watch: Tests fail, pipeline stops, email alert sent, dashboard turns red
```

**Scenario 3 — Fixed Build**
```python
# Restore the correct implementation
git add . && git commit -m "fix: restore add operation" && git push
# Watch: Pipeline recovers, re-deploys, dashboard returns green
```
